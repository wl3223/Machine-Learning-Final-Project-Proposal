# Project Checkpoint: Written Proposal

**Problem Statement**
This project utilizes the `FronkonGames/steam-games-dataset` to address the challenge of game discovery by answering the core user-facing question: "Given a game idea or preference written in plain English, what Steam games best match it, and what other games are similar?" This problem is especially interesting because Steam's massive and diverse library makes it difficult for users to find novel titles based on nuanced preferences using traditional, keyword-based search filters. By allowing users to express complex ideas naturally (e.g., "story-rich sci-fi game with controller support"), we can uncover hidden gems efficiently and provide a more intuitive, tailored browsing experience that goes beyond simple metadata matching.

**Methodology**
To enable natural-language search and game discovery, we will apply several key algorithms from the course syllabus. First, we will utilize text embeddings (via a pre-trained model like `sentence-transformers`) to convert combined text metadata—such as game descriptions, tags, and genres—into dense vector representations, effectively capturing the semantic essence of each game. We will then implement a from-scratch cosine similarity and brute-force nearest neighbor retrieval algorithm to compute the distance between the user's embedded query and the game vectors, returning top-k matches. Furthermore, to support interactive exploration of the game space, we will apply dimensionality reduction using Principal Component Analysis (PCA) to project the high-dimensional embeddings into a 2D visual map, optionally combined with K-means clustering to automatically group visually similar games into segmented neighborhoods.
